Твоя задача — подготовить полный, реалистичный и пошаговый план интеграции инфраструктурных сервисов в pet-проект Media Downloader.

ВАЖНОЕ ТРЕБОВАНИЕ:
Все инфраструктурные и прикладные сервисы должны раскатываться через `helmfile.sh`.
Не через ручной `kubectl apply`, не через отдельный `helm install`, не через `werf converge` для деплоя.
`werf` можно использовать только для сборки и публикации Docker-образов, если это уместно, но установка сервисов в Kubernetes должна выполняться через `helmfile.sh`.

Отдельно учти:
Vault конфигурируется отдельным этапом ДО инициализации всех сервисов, которым нужны секреты.
Порядок должен быть таким:

1. Поднимается Kubernetes-кластер.
2. Устанавливаются базовые CRD и namespace.
3. Устанавливается и инициализируется Vault.
4. В Vault заранее создаются секреты.
5. Устанавливается External Secrets Operator.
6. ExternalSecret-ресурсы получают креды из Vault и создают Kubernetes Secrets.
7. Только после этого инициализируются PostgreSQL, RabbitMQ, MinIO, Redis и прикладные сервисы, которые используют эти креды.

Нельзя допускать ситуацию, когда PostgreSQL, RabbitMQ, MinIO, Redis, Grafana, приложения или workers стартуют раньше, чем External Secrets успел создать нужные Kubernetes Secrets.

# Контекст проекта

Проект называется: Media Downloader.

Это DevOps pet-проект для демонстрации навыков:
- поднятие Kubernetes-кластера с нуля через Vagrant + Ansible;
- настройка containerd, kubeadm, kubelet, kubectl;
- подключение worker-нод;
- установка CNI Calico с Network Policies;
- развёртывание микросервисного приложения;
- организация CI/CD;
- GitOps-подход;
- observability;
- secrets management;
- storage;
- security hardening;
- autoscaling.

Проект представляет собой приложение для скачивания медиа из YouTube, TikTok и других платформ, последующей конвертации и выдачи результата пользователю.
